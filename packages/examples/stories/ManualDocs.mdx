# MDX

Outside of Storybook Auto Docs, you can use A11yDocs in two ways:

## with Meta `of`

Associate your MDX file with stories using the `Meta` component, then include `<A11yDocsBlock />` without any arguments to automatically use the primary story.

```tsx
import { A11yDocsBlock } from "@abcaustralia/storybook-addon-a11ydocs/storybook-addon-a11ydocs";
import { Meta } from "@storybook/addon-docs";
import * as Stories from "./Component.stories";

<Meta of={Stories} />;

<A11yDocsBlock />;
```

<div
  style={{
    background: "peachpuff",
    ...exampleStyles,
  }}
>
  <A11yDocsBlock />
</div>

## Manually

Associate your MDX file with stories using the `Meta` component, you can then include `<A11yDocsBlock />` without any arguments to automatically use the primary story.

```tsx
import { A11yDocsBlock } from "@abcaustralia/storybook-addon-a11ydocs/storybook-addon-a11ydocs";
import { Meta } from "@storybook/addon-docs";
import * as Stories from "./Component.stories";

<Meta title="Manually specified properties" />;

<A11yDocsBlock customProperties={a11ydocs.secondary} />;
```

<div style={{ background: "powderblue", ...exampleStyles }}>
  <A11yDocsBlock customProperties={a11ydocs.secondary} />
</div>
